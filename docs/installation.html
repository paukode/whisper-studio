Installation

A zero-to-running walk-through for macOS. This guide assumes no prior tooling: it installs Homebrew, Python, and the AWS CLI from scratch, then runs one script that builds and serves the app. Budget 15 to 25 minutes the first time.

Whisper Studio is one local process that serves both the API and the React SPA on a single origin. Getting there is a short chain: install a few command-line tools, connect AWS, clone the repo, run setup.sh, and configure two fields in the app. Click a box below to see what each step does.

Already tooled up? If python3 --version, brew --version, and aws --version all succeed, skip ahead to Step 4 - Get the code and run. If any of them fails, start at Step 1.

Step 1 - Install prerequisites

Four command-line tools have to be present before setup.sh can do its work. Homebrew installs the other three.

ToolWhyCheck
GitClone the projectgit --version
Python 3.10+Backend runtimepython3 --version
HomebrewInstalls everything elsebrew --version
AWS CLIBedrock auth for Claudeaws --version

First install Homebrew (skip this if brew --version already works):

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Do not skip the two eval lines. When the installer finishes it prints two eval commands that add Homebrew to your PATH. Run both, then close and reopen Terminal. Without this, the very next command reports brew: command not found.

Then install Git, Python, and the AWS CLI in one line:

brew install git python@3.12 awscli

Node.js is handled for you. You do not install Node yourself. setup.sh installs the pinned Node version inside the project venv/ using nodeenv, so it never touches a system Node or your other projects.

Step 2 - Connect AWS

Chat runs on Claude via Amazon Bedrock, so the AWS CLI needs credentials. Run:

aws configure

It prompts for an access key ID, a secret access key, a default region, and an output format. For the full walk-through of creating keys and profiles, see the AWS CLI configuration docs.

Regions must match. Whichever region you type here is where Claude must be enabled in your account. Set the same value as the Bedrock Region in Settings later. A mismatch is the most common first-run error (see Troubleshooting). You also need Bedrock model access granted for the Claude models you intend to use.

Step 3 - (Optional) Tavily key for web search

Without a Tavily key, Claude's web-search skill is unavailable; everything else (chat, transcription, the workspace tools) works. You can add this now or any time later.

  1. Sign up

    Create a key at app.tavily.com. The free tier covers personal use.

  2. Export it in your shell (recommended)

    echo 'export TAVILY_API_KEY=tvly-...' >> ~/.zshrc
source ~/.zshrc

    The env var overrides any value stored in config.json, so rotating the key is one export away and the secret stays out of the config file.

  3. Or paste it into Settings

    You can instead paste the key into the Settings UI. If both are set, the environment variable wins.

See Environment variables for the full list of what the shell layer can override.

Step 4 - Get the code and run

There are two ways to get the project onto your machine. Both finish with one command that builds and serves the app, so pick whichever suits you.

Option A - Clone with Git (for developers)

Clone the repository, enter it, and run the setup script:

git clone https://github.com/paukode/whisper-studio.git
cd whisper-studio
bash setup.sh

Option B - Download the ZIP (no Git needed)

If you would rather not touch the command line to get the code, download the project as a ZIP and run the same script.

  1. Download

    On the GitHub repository page, use Code → Download ZIP and save it (for example to your Downloads folder).

  2. Extract

    Double-click the downloaded .zip in Finder. It expands into a whisper-studio folder.

  3. Run setup.sh

    Open that folder, right-click setup.sh, and choose Open With → Terminal. The script runs and prints the URL when the app is ready. That is it - you are good to go.

If double-clicking opens a text editor instead of running it, macOS has not associated .sh files with Terminal. Use the right-click → Open With → Terminal path above, or open Terminal, type bash (with a trailing space), drag the setup.sh file onto the window, and press Enter. On the first run, macOS Gatekeeper may ask you to confirm running a downloaded script.

From here on both options are identical - the rest of this guide applies either way.

On the first run the script does the full provisioning pass:

  1. Checks Python

    Requires Python 3.10 or newer (the backend uses match statements and asyncio.TaskGroup). If the active Python is older or missing, it offers to install Python 3.12 via Homebrew.

  2. Creates the venv and installs Python deps

    Builds an isolated venv/ and installs from requirements.txt. It prefers a Homebrew Python that can load SQLite extensions so vector search can use sqlite-vec, falling back to a numpy vector search otherwise. Your default python3 is left untouched.

  3. Installs system tools

    Installs ffmpeg (audio decoding for Whisper) and ripgrep (workspace search) via Homebrew, plus the Playwright Chromium build used by live preview.

  4. Installs the pinned Node inside the venv

    Uses nodeenv to install the Node version from .nvmrc into the venv, then runs npm install.

  5. Builds and serves the frontend

    Runs npm run build, then starts the FastAPI backend, which serves the compiled SPA. It picks a free port starting at 8000, waits for the backend to answer /health, and prints the URL. Press Ctrl+C to stop.

On the first run the script also seeds config.json from config.example.json and downloads model weights into models/. Transcription weights (Whisper, Parakeet, and the speaker encoder) are always pulled; the on-device index and chat weights are only pulled for hybrid and local modes (see Model modes). Every download is idempotent and non-fatal: a pull that cannot finish now retries automatically on first use.

Subsequent runs are fast. When venv/ and node_modules/ already exist, the install steps are skipped. A plain bash setup.sh just rebuilds the frontend and serves it.

setup.sh flags

Flags fall into three groups: what to reinstall, how to serve, and which model mode to provision. With no flags you get production serving in the existing (or cloud) mode.

FlagEffect
--newWipe venv/, node_modules/, and static/dist/, then reinstall from scratch. It also backs up the live config.json (timestamped) and reseeds it from config.example.json, so re-add your Tavily key and any custom settings afterwards, or restore them from the backup.
--devDevelopment mode: run the backend plus the Vite dev server on port 5173 with hot-module reload. Open the Vite URL, not the backend.
--prodProduction mode (the default): build the frontend and serve it from FastAPI on port 8000. No Vite, no HMR. Mutually exclusive with --dev.
--cloudCloud mode (the default for a fresh install): all indexing and RAG on Amazon Bedrock. Skips the ~16 GB of on-device weights.
--hybridHybrid mode: pick a backend per capability. Pulls the on-device weights so you can choose them.
--localLocal mode: everything on-device. Pulls all on-device weights.
--no-openDo not auto-open the browser (also honored via the NO_OPEN=1 env var). Useful for SSH or headless sessions.
--helpShow all options and exit.

Without a mode flag, an existing config.json is honored; a fresh install defaults to cloud. The --dev/--prod and mode flags can be combined, for example bash setup.sh --dev --local.

Model modes

Model mode controls where indexing and retrieval run. It is separate from your chat-model choice, which lives in the toolbar. You can set the mode at install time with the flags above, or change it any time under Settings → Model Mode (changing it prompts a reload so the app re-initializes).

ModeEmbeddingsRerankerEntity extractionOn-device weights
Cloud (default)Cohere Embed v4 (Bedrock)Cohere Rerank 3.5Claude Haikunone - lean install
Hybridper-capability pickper-capability pickper-capability pickpulled
LocalQwen3 (on-device)Qwen3 RerankerGLiNER~16 GB, all on-device

Each embedder keeps its own index. Switching modes never rebuilds the other backend's index, so you re-index a folder once under a given backend to search it there. Cohere embed and rerank are pinned to us-east-1; chat can run in any region. Transcription is always on-device (Whisper/Parakeet) in every mode.

For the full picture of how each backend is wired, see Model modes and the architecture page Model backends & modes.

Step 5 - Configure in the app

Open the URL the script printed (something like http://127.0.0.1:8000), click the gear icon, and fill in:

Bedrock Region
The same region you used in aws configure, where Claude is enabled.
Default Model
Start with Sonnet - a good balance of speed and quality.
Tavily API Key
Only if you did not set the TAVILY_API_KEY environment variable in Step 3.

Save, then type hello in chat. If Claude responds, the install is verified.

Transcription needs no setup. The speech model (Whisper/Parakeet) auto-downloads on your first recording (or was pulled during setup) and runs entirely on your machine. There is nothing to configure to record.

Where settings live

config.json at the repo root is gitignored and holds your per-machine settings (your Tavily key, region, default model, and more). Because it is gitignored, a fresh clone has none, so on the first run setup.sh copies config.example.json to config.json for you. After that you change values either through the Settings UI or by editing the file directly. Environment variables override the file, and a per-project .whisper/settings.json sits between them. See Configuration for the full three-layer model.

Troubleshooting

Most first-run problems fall into a handful of buckets. Match the symptom, apply the fix, re-run.

SymptomFix
python3: command not foundRun brew install python@3.12, then reopen Terminal.
brew: command not foundRun the two eval lines the Homebrew installer printed, then reopen Terminal.
Python 3.x detected; this project requires Python 3.10 or newerRun brew install python@3.12, then reopen Terminal so the updated PATH applies.
setup.sh: Permission deniedRun it as bash setup.sh, not ./setup.sh.
npm install network errorsYou are behind a proxy or firewall. Try another network, or npm config set registry https://registry.npmjs.org/.
AccessDeniedException / Could not invoke model in chatThe AWS user lacks Bedrock permissions, or the Claude model is not enabled in your region.
You don't have access to the model with the specified model IDRegion mismatch - the model is enabled in a different region than Settings says. Align the Bedrock Region.
Port 8000 or 5173 already in useThe script auto-increments to the next free port. Read the URL it actually prints.
Blank page in the browserThe frontend bundle is missing. Re-run bash setup.sh, or bash setup.sh --new for a clean rebuild.

When in doubt, run /doctor. Type it in the chat input for an automated dependency and AWS-access check that reports exactly what is missing or misconfigured.
