Glossary

The vocabulary of Whisper Studio, defined the way this project uses each term. Where a concept has its own page, the entry links to it.

Terms are grouped alphabetically. Many of these words have broader meanings in the wider industry; the definitions here describe how Whisper Studio specifically uses them. When in doubt, the linked architecture page is the source of truth.

Looking for something specific? Press / (or click the search box in the sidebar) to filter every page, including this one, by term.

A - B

TermDefinition
ASR Automatic Speech Recognition. The task of turning audio into text. Whisper Studio runs ASR entirely on-device (Whisper or Parakeet) in every model mode, so audio never leaves your machine. See Transcription & diarization.
Bedrock Amazon Bedrock, the AWS managed service Whisper Studio uses to call Claude (and GPT-5.x) for chat. Requests go from the local process straight to bedrock-runtime.<region>.amazonaws.com and back. It is the only cloud LLM call the app makes. See Model backends & modes and the Amazon Bedrock docs.
BlackHole A macOS virtual audio driver used to route system audio into a recording. Paired with an Aggregate Device, it lets a meeting recording capture both your microphone and whatever macOS is playing. Install with brew install blackhole-2ch. See Voice & meetings.

C

TermDefinition
chat turn One request/response cycle: your message plus everything the assistant does to answer it (streamed text, thinking, tool calls, approvals) up to [DONE]. A single turn is one POST /api/chat that streams back over SSE and may run an agentic tool loop of up to 50 rounds. See Anatomy of a chat turn.
cron_event A chat-history row produced by a scheduled (cron) task. Cron output cards persist on disk and render in the chat, but visible_chat_history filters them out before the next prompt is built, so a job firing frequently never inflates the token bill. See Cron / scheduled tasks.

D - E

TermDefinition
detached / worktree agent An agent started in the background rather than inline. spawn_agent with detach: true returns a task_id at once (read-only by default) so the turn is not blocked; isolation: 'worktree' additionally runs it inside an isolated git worktree so its edits never touch your current checkout. Complements the sub-agent and worktree entries. See Sub-agents & the event bus.
diarization Assigning "who spoke when" to a transcript. Whisper Studio uses Resemblyzer speaker embeddings plus voice-activity detection, and remembers speakers per session across reconnects so you can rename them once. See Transcription & diarization.
DOMPurify The HTML sanitizer (version 3.4.6) that every Markdown render passes through before it reaches the React DOM. It strips <script>, inline event handlers, and javascript: URLs, so a prompt-injected page fetched by web_fetch cannot run arbitrary JS in the SPA origin. See Security model.
effort level How much reasoning budget the model spends per turn: low, medium, high, extra (the raw name xhigh is an alias), max, or ultracode. The former auto level is retired (coerced back to a real level). ultracode is a session mode that pairs top reasoning effort with dynamic-workflow orchestration. Set it with the /effort command or in Settings. See Settings & config keys and The ultracode workflow runtime.
embedding A numeric vector that represents the meaning of a chunk of text so similar passages sit close together. Used by the index to find relevant context for grounding. The embedder depends on model mode: Cohere Embed v4 in cloud, Qwen3 on-device in local. See Index / GraphRAG & search.
entity A named thing (person, file, project, concept) extracted from your indexed content and wired into the knowledge graph. Entities are extracted by Claude Haiku in cloud mode and by GLiNER on-device. See Index / GraphRAG & search.
event bus The per-session publish/subscribe channel (server/agents/event_bus.py) that fans sub-agent and team activity out to the browser over SSE. Spawns, progress, and completions all travel across it. See Sub-agents & the event bus.

G

TermDefinition
GLiNER An on-device model used for entity extraction when the app runs in local mode (the cloud equivalent is Claude Haiku). It feeds the knowledge graph without any network call. See Index / GraphRAG & search.
goal / goal gate A per-session objective the assistant works toward. Set it with /goal (and /goal clear to end it); it is stored on the session row and a goal banner shows above the composer. After each turn a completion gate - a cheap Haiku evaluator in server/goals/gate.py - judges whether the goal is met, and if not the goal loop auto-continues the turn (capped). Gated by the goal_loop feature flag. See The goal loop & completion gate.
GraphRAG Retrieval-augmented generation that combines vector search over embeddings with a graph of entities and their relationships, rather than plain nearest-neighbour lookup. It powers workspace indexing and the grounding chip shown during a turn. See Index / GraphRAG & search.
grounding Injecting passages retrieved from a selected workspace index into the prompt so the answer is anchored to your own documents. A "grounding" chip over SSE reports which folders were searched and how many passages were retrieved. See Index / GraphRAG & search.

H - L

TermDefinition
hook A user-defined gate that runs at a fixed point in a turn: PreToolUse (can deny or rewrite a tool call), PostToolUse (can inject additional context), and a Stop gate, wired across chat, sub-agents, and cron (server/hooks/). User hooks live in hooks.json; a workspace may also ship trust-gated project hooks in .whisper/settings.json that stay inert until explicitly trusted. Managed in Settings, not from a slash command. See Blocking hooks.
hybrid mode One of three model modes. In hybrid mode you pick a backend per capability (embeddings, reranker, entity extraction), mixing cloud and on-device. It pulls the on-device weights so those options are available. Compare cloud (the default) and local. See Model modes and Model backends & modes.
LSP Language Server Protocol. Whisper Studio proxies python-lsp-server and typescript-language-server to power the lsp_diagnostics, lsp_hover, and lsp_references tools, giving the assistant real type information about your code. See Git integration & LSP.

M - N

TermDefinition
Map-reduce condensation Splitting an oversized transcript into overlapping chunks, extracting the salient material from each with a fast one-shot model (the map step), then letting the main chat model compose the summary from the concatenated extracts (the reduce step). Extracting rather than prose-summarising is what keeps owners, decisions, and numbers intact where a plain truncation would lose them. See Transcript summarization & condensation.
MCP Model Context Protocol. A standard for exposing tools from external systems. Whisper Studio spawns configured MCP servers as child processes piped over stdio, so they add tools without ever seeing HTTP. Configure servers in Settings and enable them per turn. See MCP & plugins.
model mode Where indexing and retrieval run: cloud (default), hybrid, or local. This is separate from the chat-model choice in the toolbar. Cohere embed/rerank are pinned to us-east-1; chat can run in any region. See Model modes.
nonce A single-use token the server holds to authorize one risky action. When a tool needs consent, the approval router issues a nonce, pauses the turn, and only runs the action once the client returns that exact nonce. The client cannot fabricate one, which is why command approval is a real boundary and not client-asserted. See Approval & permissions.

O

TermDefinition
One-shot completion A single non-streaming prompt and response, used by the map step of map-reduce condensation. It is mode-aware - Claude Haiku in the cloud, the resident model on-device - and raises on failure so the condenser can fall back to a truncated transcript instead of returning nothing. See Transcript summarization & condensation.

P - Q

TermDefinition
Parakeet One of the on-device speech-recognition models Whisper Studio can use for transcription (the other is Whisper). Which one runs depends on your setup; both keep audio local. See Transcription & diarization.
permission mode The dial that decides how tool calls are gated: default (reads free, writes ask), auto (a classifier decides per call), plan (read-only), acceptEdits (edits auto-approve), bypassPermissions (no prompts), or dontAsk (reads allow, writes silently deny). Per-tool rules override any mode. See Permissions & approvals.
plugin An opt-in Python file in plugins/ that adds custom API routes and tool executors. Dropping a file in does nothing until you enable it in Settings and restart; this closes the drive-by "plugin over chat" attack vector. See MCP & plugins.
progressive tool disclosure Sending the model a core tool set plus any session-activated tools, with everything else represented only by a compact index in the cached prompt block; deferred tools are pulled in on demand via tool_search (server/chat/tool_partition.py). This cuts roughly 60-75% of the tool-schema tokens per turn. Gated by the progressive_tools feature flag. See Settings & config keys.
Qwen3 The on-device model family used for embeddings and reranking in local mode (Qwen3 embedder and Qwen3 Reranker). It lets indexing and retrieval run without any cloud call. See Index / GraphRAG & search.

R - S

TermDefinition
reranker A second-pass model that reorders retrieved passages by relevance to the query, sharpening what actually reaches the prompt. Cohere Rerank 3.5 in cloud mode, Qwen3 Reranker on-device. See Index / GraphRAG & search.
sandbox-exec The macOS kernel-level sandbox (sandbox-exec(5)) that wraps every shell and code-execution path, including the terminal PTY. It is the security boundary the project trusts last and most; a curated deny-list masks secret stores (SSH/GPG keys, cloud and git credentials, browser cookies, shell history). On Linux the equivalent is bwrap. See Sandbox & security.
skill A Markdown-defined capability the assistant loads when relevant (Web Search, Run Python, Summarize Transcript, and more). Built-in skills load automatically; add your own as files in skills/. Invoke or list them with skill_invoke and skill_list. See Skills.
SSE Server-Sent Events. The one-way HTTP streaming used to push chat tokens, tool activity, usage counters, and sub-agent events to the browser as they happen. A chat turn is an SSE stream, not a single JSON body, and it ends with [DONE]. See Chat & tool orchestration.
sub-agent An independent agent spawned to handle a focused subtask, coordinated through the event bus and optionally organized into a team of parallel agents. Spawn one with spawn_agent or the /subagent command. See Sub-agents & the event bus.

T - V

TermDefinition
task registry / background task The unified store of long-running background jobs - shell commands, agents, workflows, and CI watches - in the agent_tasks table (server/tasks/). Exposed at /api/background-tasks and surfaced as a running-count pill in the header; it replaced the older server/background_tasks.py. See Sub-agents & the event bus.
Tavily The web-search provider behind the web_search skill. It is optional: without a TAVILY_API_KEY, web search is unavailable and everything else still works. The env var overrides any key stored in config.json. See Web research and app.tavily.com.
Transcript condensation The server-side step that shrinks a transcript too large for the model context before it enters a chat turn, so a summary skill never silently truncates or overflows the request. See the Map-reduce condensation entry and Transcript summarization & condensation.
ultracode Whisper Studio's top reasoning mode: the model authors a deterministic JavaScript workflow script that a Node-24 vm harness runs detached over stdio JSON-RPC into Python, on both Claude and GPT-on-Bedrock, with 16-way agent concurrency, a 1000-agent lifetime cap, USD budgets, a resumable journal, and upfront phase-preview approval of every new script. It is also the highest /effort level. See The ultracode workflow runtime.
VAD Voice-Activity Detection. Distinguishes speech from silence in the audio stream, feeding diarization and keeping the transcript clean. Whisper Studio uses WebRTC VAD locally. See Transcription & diarization.

W - Z

TermDefinition
WAL Write-Ahead Logging, the SQLite journaling mode Whisper Studio runs sessions in. It lets reads and writes proceed concurrently, which matters when a background cron daemon appends rows while the frontend saves. See Persistence.
WHISPER.md A project-context file you drop in your workspace root. Its contents load into the system prompt on every chat turn, the way CLAUDE.md works for the Claude Code CLI. Edit it through the in-app Project Memory dialog or any editor. See Memory & WHISPER.md.
Whisper The on-device speech-recognition model that gives the app its name (Whisper Studio runs the mlx-whisper build). It loads into memory on first record and runs locally; Parakeet is the alternative. See Transcription & diarization.
workflow runtime The subsystem behind ultracode (server/workflows/): a WorkflowRun that enforces the concurrency semaphore, USD budget, and resume; the Node-24 harness (harness.mjs, guards.mjs, api.mjs); the workflow_runs table; and the workflow_run / workflow_status / workflow_save / workflow_list tools, available only in ultracode mode. See The ultracode workflow runtime.
workspace The folder you connect so the assistant can read, edit, run, and commit within it. Every file operation is validated to stay inside the workspace root, and project files like WHISPER.md and .whisper/ live there. See The workspace IDE.
worktree An isolated Git working tree on its own branch, created so the assistant can make changes without touching your current checkout. Managed with ws_create_worktree, ws_diff_worktree, and ws_merge_worktree. See Git integration & LSP.

Whisper (the speech model) and Whisper Studio (the app) are different things. When these docs say "Whisper" alone in a transcription context, they mean the model; the product is always "Whisper Studio".
