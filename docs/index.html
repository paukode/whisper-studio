Quick guideline

If you already have Python 3.10+, Homebrew, Git, and the AWS CLI configured, you are four steps from a running app. The setup script provisions everything else in an isolated venv/.

  1. Clone the repository

    git clone https://github.com/paukode/whisper-studio.git
cd whisper-studio

  2. Run the setup script

    It creates the venv, installs Python + Node deps, builds the frontend, and serves it.

    bash setup.sh

  3. Open the URL it prints

    Something like http://127.0.0.1:8000. Click the gear icon and set your Bedrock Region and Default Model.

  4. Say hello

    Type hello in chat. If Claude responds, you are done. Transcription needs no setup - the speech model (Whisper/Parakeet) auto-downloads on your first recording.

New here? The Installation guide assumes zero prior tooling and walks through Homebrew, Python, and AWS from scratch.

Highlights

Local transcription

Two on-device engines: Parakeet streams words in real time (low latency); Whisper transcribes each settled utterance in a batch. Both run locally with speaker diarization, and audio never leaves the laptop.

Chat with Claude

Streaming tokens, tool use, attachments, slash commands, and voice-submit, via Amazon Bedrock (Claude or GPT-5), or an on-device model in local mode.

Full workspace IDE

File tree, Monaco editor, xterm terminal, Git, LSP, and ripgrep search in the same tab.

Consent by design

A nonce-based approval flow and an OS sandbox gate every risky action. Reads run freely.

Background jobs

A unified background-task registry (kinds shell, agent, workflow, and ci) surfaced by a header running-count pill and a status-bar task pill. Cron-style scheduled tasks are one kind; their output streams back into the chat that created them.

Ultracode workflows

Model-authored deterministic JS orchestration, run detached in a Node-24 vm harness over JSON-RPC into Python - on Claude and GPT-on-Bedrock - with 16-way concurrency, USD budgets, a resumable journal, and upfront phase-preview approval. How it works →

Guardrails and autopilot

Blocking hooks gate every tool call (Pre/Post/Stop), a goal loop keeps a turn running until a completion gate clears it (/goal), and CI watch plus PR autofix chase a red build back to green (/ci). Get started →

Extensible

Cloud / hybrid / local model modes, MCP servers, opt-in Python plugins, and custom skills.
